## 💡 Filosofia

<div align="center">

> *"Qualquer tolo pode escrever código que um computador entende.*
> *Bons programadores escrevem código que humanos entendem."*
>
> — **Martin Fowler**

</div>

<br/>

```
  PRINCÍPIO              APLICAÇÃO PRÁTICA
  ──────────────────────────────────────────────────────────────
  Single Responsibility  →  Cada classe tem UMA razão para mudar
  Open / Closed          →  Extensível sem modificar o existente
  Liskov Substitution    →  Subtipos se comportam como esperado
  Interface Segregation  →  Contratos coesos, sem gordura
  Dependency Inversion   →  Dependo de abstrações, nunca de detalhes
  ──────────────────────────────────────────────────────────────
  DRY   →  Não me repito. Jamais.
  KISS  →  Simples funciona. Simples escala. Simples vence.
  YAGNI →  Não construo o que não é necessário hoje.
  ──────────────────────────────────────────────────────────────
```
